# Surfing 작업 정리

## 개요
- 기존 일정/신청 흐름을 랜딩 중심의 홈 화면으로 재구성했다.
- `/meeting/[id]` 중심 흐름을 줄이고, 홈에서 달력 선택, 상세 확인, 신청, 신청현황 확인이 가능하게 바꿨다.
- 프로필 편집, 커스텀 프로필 이미지, 참가 옵션 설정, 관리자 설정까지 연동했다.

## 홈/랜딩
- 메인 페이지 `/`를 Stitch 기반 랜딩 화면으로 교체했다.
- 달력은 해당 월의 모든 주를 렌더링하도록 수정했다.
- 초기 진입 시 오늘 기준 가장 가까운 모임이 선택되도록 변경했다.
- 빈 날짜를 선택하면 모임상세와 파도예보를 노출하지 않도록 정리했다.
- `/meeting/[id]`는 `/?date=YYYY-MM-DD`로 리다이렉트되게 변경했다.

## 모임 상세/신청
- 모임상세를 `참가하기`, `신청현황` 탭 구조로 변경했다.
- 탭은 캘린더 바로 아래에 full-width 언더라인 형태로 배치했다.
- `신청현황` 탭에는 참가자 수 badge를 표시하고 9명 이상은 `9+`로 표기한다.
- 파도예보 하단 중복 섹션을 없애고 상세 내부 구조를 정리했다.
- 신청현황 리스트는 동반인 시스템 메모(`~~의 동반`)를 숨기고, 번호를 전체 순번 기준 숫자로 표시한다.
- 비고가 없는 행은 숫자와 이름이 세로 가운데 정렬되도록 수정했다.

## 참가 옵션
- `내 참가 옵션`에 가격 안내 info 버튼과 툴팁을 추가했다.
- 참가 옵션 가격 안내는 관리자 설정에서 수정 가능하게 만들었다.
- 참가 옵션 가격 안내 공개 읽기용 API `/api/settings/public`를 추가했다.
- 옵션 구조를 다음과 같이 정리했다.
  - 배타적 옵션: `강습+장비대여`, `장비 대여만`
  - 추가 옵션: `셔틀 버스`
- `강습+장비대여`와 `장비 대여만`은 동시에 선택되지 않도록 처리했다.
- 세 옵션은 모두 같은 위계의 체크박스 형태로 맞추고, `강습+장비대여`와 `장비 대여만`만 내부 로직에서 배타적으로 처리했다.
- 같은 구조를 본인 신청, 동반인 선택, 신규 동반인, 이미 신청된 동반인 수정에 모두 적용했다.
- 신청현황/관리자 화면의 옵션 칩 라벨도 동일하게 맞췄다.

## 프로필
- 카카오 기본 프로필 외에 서비스 전용 커스텀 프로필 이미지 업로드 흐름을 추가했다.
- 브라우저에서 먼저 리사이즈/압축한 뒤 업로드하는 방식으로 구현했다.
- 프로필 페이지 헤더를 메인과 같은 상단 바 구조로 재정리했다.
- 프로필 사진 박스를 제거하고, 상단 아바타 우하단 카메라 버튼으로 즉시 업로드되도록 바꿨다.
- 프로필 화면은 `기본 정보`와 `동반인 관리` 탭으로 나눴고, 정회원 표시는 회색 톤으로 정리했다.
- 홈 헤더에서는 커스텀 프로필 이미지가 있으면 우선 노출되게 변경했다.

## 인증/로그인
- 카카오 로그인 redirect URI 처리와 호스트 불일치 문제를 정리했다.
- `layout.tsx`의 raw `<script>`를 `next/script`로 교체해 클라이언트 렌더링 에러를 해결했다.
- 프로필 진입 시 세션은 있으나 유저 row가 없는 경우에도 자동 `upsert`되도록 보강했다.

## 관리자
- 관리자 설정 페이지에 취소 패널티 설정을 유지했다.
- 관리자 설정 페이지에 `참가 옵션 가격 안내` 편집 UI를 추가했다.
- 관리자 모임 상세에서 참가자 옵션 칩을 홈 화면과 같은 용어로 맞췄다.

## 데이터/DB
- `Participant.hasRental` 필드를 추가해 `장비 대여만` 옵션을 저장하도록 확장했다.
- 로컬 개발 DB에는 `prisma db push`로 스키마를 반영했다.
